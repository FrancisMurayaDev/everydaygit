EveryDay Git

1.What is Git

Git is a free, open-source version control system (VCS) that helps developers track and manage changes to their code. Git is a widely used, distributed version control system that allows developers to track changes made to code over time, enabling them to revert to previous versions, collaborate effectively, and manage different variations of a project simultaneously, all by maintaining local copies of the codebase and periodically syncing them with a central repository.

A version control system (VCS) is a software tool that allows developers to track and manage changes made to files over time, essentially keeping a record of every modification to a project, including who made the change, when it was made, and what specific alterations were done, enabling collaboration on the same files without conflicts within a team.

Other control versions like Git include Apache, Perforce and Subversion. However, Git stands out due to its unique features, flexibility and wide spread adoption.

3.Configuring Git

The following is the process of configuring the username of your git.

  1. Open Command Prompt
  2. Run the Command: Git config --global user.name “USERNAME”

The following is the process of configuring the email address of your git.

  1. Open command Prompt
  2. Run the command: Git config --global user.email “EMAIL”

4.Initializing a Git Repository

A Git repository is a centralized location where all versions of a project's files are stored and tracked, allowing developers to manage changes to their code over time, essentially creating a history of edits within a project, all managed by the Git version control system; it is essentially a folder within a project containing a hidden ".git" directory which holds all the versioning information about the files within that project.

To Initialize a Git Repository:

  • Open the terminal or the command prompt in your project directory, you can also open your command line tool and navigate to your project directory.
  • Run the command: git init

The command will initialize a git repository in the project folder, this is where git will track and manage changes to files in your project. Initializing a git is done once per project. Running git init in a project that contains a git repository will make git initialize a repository within a repository.

5.Adding Files to the Staging Area

In Git, the staging area also known as the index is an intermediate space where you can organize changes before committing them to the repository.

Tracked files are files that Git knows about. Untracked files are everything else — any files in your working directory that were not in your last commit and are not in your staging area.

Commands Used in the Staging Area

  • git add command to add files in the staging Area.
  • Write the command git add and the file name.

  • To add all files in one go, we use the command git add . or git add -A.

6.Commiting Changes

A commit in git is basically a snapshot of your project at a particular point in time. Commits work when there are changes in the staging area. A good commit message should be written in present tense. One should also ensure the commit message sufficiently explains the code.

Committing changes in a git repository.

  • Open terminal, run the command git status to see the changes to be commited. Enter git commit -m”Commit Message”
  • To view past commits we the command git log

7.Branches

In Git, a "branch" is essentially a pointer to a specific commit within a repository allowing developers to work on different versions of code simultaneously without affecting the main codebase; it acts like a separate line of development that can be created, merged, and deleted as needed, enabling parallel work on features or bug fixes.

8.Creating a Branch

To create a branch we use the command git branch BRANCH_NAME

9.Viewing all Branches

To view the existing branches in a repository, use the commands git branch. The active branch will have a little asterisk symbol (*) besides it and it will be green in some command line interfaces.

10.Switching Active Branches

To switch branches, we use the command git switch Branch_Name

11.Merging a Branch

First switch to the branch we want to merge the current branch. For instance we start by switching to the main branch. use the command git merge BRANCH_NAME

12.Deleting a Branch

To delete a branch, we use the git branch command and pass the -d flag.

Git branch -d BRANCH_NAME
